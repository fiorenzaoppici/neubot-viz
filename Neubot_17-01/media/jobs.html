    Jobs

    Jobs are objects that manage asynchronous tasks such as exporting data, importing a table, and running a query.

    You can run multiple jobs concurrently in BigQuery, and completed jobs will be listed in the Jobs collection indefinitely unless you call bigquery.jobs.delete() on a job.

    For a list of methods for this resource, see the end of this page.

    Resource representations

    Template for a Job resource.

    {
          "kind": "bigquery#job",
          "etag": etag,
          "id": string,
          "selfLink": string,
          "jobReference": {
            "projectId": string,
            "jobId": string
          },
          "configuration": {
            "load": {load config},
            "extract": {extract config},
            "copy": {copy config},
            "query": {query config}
          },
          "status": {
            "state": string,
            "errorResult": {
              "reason": string,
              "location": string,
              "debugInfo": string,
              "message": string
            },
            "errors": [
              {
                "reason": string,
                "location": string,
                "debugInfo": string,
                "message": string
              }
            ]
          },
          "statistics": {
            "startTime": long,
            "endTime": long,
            "totalBytesProcessed": long,
            "query": {
              "totalBytesProcessed": long
            },
            "load": {
              "inputFiles": long,
              "inputFileBytes": long,
              "outputRows": long,
              "outputBytes": long
            }
          }
    }
    Property name Value Description
    kind bigquery#jobs The type of resource. Always bigquery#jobs.
    etag etag [Output-only] A hash of this resource.
    id string A string that uniquely identifies this job across projects. Do not use this in methods that require a job ID; use the jobReference value instead.
    selfLink string An URL that can be used to access this resource again. You can use this URL in get() requests for this resource.
    jobReference nested object Identifying information about this job.
    jobReference.projectId string The ID of the project being billed for this job.
    jobReference.jobId string A unique ID for this job within the project. This value is optional when you create a new job; if you do not specify a value here, a random ID will be chosen for you. It must a string of 1-1,024 characters satisfying the regular expression [a-zA-Z][\w]{0,1023}, where \w is any digit, underscore, or upper or lowercase letter.
    configuration nested object

    An object that specifies the details of this job. When inserting a new job, include only one of the following child objects, depending on what type of job this is.

    • load - Create and populate a new table from a CSV file
    • query - Run a query
    • extract - Export a BigQuery table to Google Cloud Storage as a CSV file
    • copy - Copy an existing table
    status nested object [Output-only] The status of this job. Examine this value when polling an asynchronous job to see if the job is complete.
    status.state string

    The current state of this job. Supports the following string values: 

    • PENDING - Queued
    • RUNNING - Running.
    • DONE - Completed, either successfully or not. If unsuccessful, the errorResult field should contain additional information.
    status.errorResult nested object An object that will only be present if the job has failed.
    status.errorResult.reason string Specifies the error reason. For example, reason will be "required" or "invalid" if some field was missing or malformed.
    status.errorResult.location string Specifies where the error occurred, if present.
    status.errorResult.message string A user-friendly description of the error. If you want to trap specific errors, use status.errors.code.
    status.errors[] list A list of non-fatal errors that occurred during job processing. It is possible that a job can complete successfully even if there are some errors. In import or export requests, it is possible that some rows were unable to be imported or exported due to errors.
    status.errors[].reason string A BigQuery error code appropriate for this error. See Handling Errors.
    status.errors[].location string Specifies where the error occurred, if present.
    status.errors[].message string A user-friendly description of the error. If you want to trap specific errors, use status.errors.code.
    statistics nested object [Output-only] Information about the job, including starting time and ending time of the job.
    statistics.startTime long Start time of this job, in milliseconds since the epoch. This starts ticking when the job status changes to RUNNING, not when the request is sent by the user or received by BigQuery.
    statistics.endTime long End time of the job, in milliseconds since the epoch. This is when the status changed to DONE, successfully or not. If the job has not finished, this property will not be present.
    statistics.totalBytesProcessed long [Output-only] [Deprecated] Use the bytes processed in the query statistics instead.
    status.errorResult.debugInfo string Debugging information for the service, if present. Should be ignored.
    status.errors[].debugInfo string Debugging information for the service, if present. Should be ignored.
    statistics.query nested object [Output-only] Statistics for a query job.
    statistics.query.totalBytesProcessed long [Output-only] Total bytes processed for this job.
    statistics.load nested object [Output-only] Statistics for a load job.
    statistics.load.inputFiles long [Output-only] Number of source files in a load job.
    statistics.load.inputFileBytes long [Output-only] Number of bytes of source data in a joad job.
    statistics.load.outputRows long [Output-only] Number of rows imported in a load job. Note that while an import job is in the running state, this value may change.
    statistics.load.outputBytes long [Output-only] Size of the loaded data in bytes. Note that while an import job is in the running state, this value may change.

    Load Config

       "load": {
              "sourceUris": [
                string
              ],
              "schema": {
                "fields": [
                  {
                    "name": string,
                    "type": string,
                    "mode": string,
                    "fields": [
                      (TableFieldSchema)
                    ]
                  }
                ]
              },
              "destinationTable": {
                "projectId": string,
                "datasetId": string,
                "tableId": string
              },
              "createDisposition": string,
              "writeDisposition": string,
              "fieldDelimiter": string,
              "skipLeadingRows": integer,
              "encoding": string,
              "quote": string,
              "maxBadRecords": integer,
              "allowQuotedNewlines": boolean,
              "sourceFormat": string
            }
    Property Name Value Description
    configuration.load nested object An object that must be present only when importing data into a new or existing table from a CSV data file.
    configuration.load.sourceUris[] list A list of one or more Google Cloud Storage objects containing table data. The person calling jobs.insert must have read access to all objects referenced. All objects must be uncompressed or gzipped CSV files in the proper format, with the same table schema. These must be fully-qualified names, for example: gs://mybucket/myobject.csv
    configuration.load.schema nested object A schema descriptor that describes the data. Only required if the table has never had a schema assigned. See Data Format for a description of the supported file formats, data types, modes, and so on for a table.
    configuration.load.schema.fields[] list A list of one or more objects, each describing a column in the imported table(s) schema. These values will be applied to the table created, if a new table is created rather than appended to.
    configuration.load.schema.fields[].name string A friendly name for this column. This name will be assigned to the column in the newly created table.
    configuration.load.schema.fields[].type string The field type.
    configuration.load.schema.fields[].mode string The field mode. Default is NULLABLE.
    configuration.load.schema.fields[].fields[] list Present only in a column that holds nested fields. This object describes any nested fields.
    configuration.load.destinationTable nested object The tableID of the destination table to hold the loaded data. Specify whether to create a new table or overwrite or append to an existing one by setting createDispostion and writeDispostion
    configuration.load.destinationTable.projectId string The ID of the project that contains the dataset to write the result table to. If not specified, will write to the project containing the job.
    configuration.load.destinationTable.datasetId string The unqualified ID of the dataset to write the result table to. You must have write access in this dataset.
    configuration.load.destinationTable.tableId string The unqualified table ID of the table to create or append to. The table must already exist unlessconfiguration.load.createDisposition=CREATE_IF_NEEDED
    configuration.load.createDisposition string

    Whether or not to create a new table, if none exists. The following string values are supported:

    • CREATE_NEVER - Do not create a new table.
    • CREATE_IF_NEEDED - [Default] If a table does not exist, create one.
    configuration.load.writeDisposition string

    Whether or not to overwrite an existing table. The following string values are supported:

    • WRITE_EMPTY - [Default] Only write to a table with no data, otherwise fail.
    • WRITE_TRUNCATE - If a table exists, replace existing data with new data.
    • WRITE_APPEND - If a table exists, append new data to the end.
    configuration.load.fieldDelimiter string The field delimiter in the source file. By default, this is a comma. You can specify any ASCII character.
    configuration.load.skipLeadingRows integer How many of the top rows to skip when importing a CSV file into a table. This is useful for skipping any header rows in the CSV file. Default value is 0 (all rows considered data rows).
    configuration.load.encoding string The encoding of the source data. This is useful if you're trying to import ISO-8859-1 (or Latin-1) encoded data and are having problems with your imported data (e.g. if your data is displaying invalid characters). You can use this parameter to let BigQuery know that your data is encoded in ISO-8859-1 or UTF-8 and BigQuery converts the data to UTF-8 if necessary. The supported values for this parameter are:

    • UTF-8 - This is the default encoding. If you do not specify this parameter, it is assumed that the data is UTF-8 encoded.
    • ISO-8859-1 - If your data is encoded in ISO-8859-1 (or Latin-1), specify this value.
    configuration.load.maxBadRecords integer How many bad records to ignore before the entire import job is aborted and no updates are performed. If this value is larger than 0, the job will succeed as long as the number of bad records do not exceed this value. This is useful if you would like to import files that may have bad records. The default value for this parameter is 0 (all rows are required to be valid).
    configuration.load.quote string [Optional] Quote character to use. Default is '"'. Note that quoting is done on the raw, binary data before the encoding is applied. If no quoting is done, use am empty string.
    configuration.load.allowQuotedNewlines boolean [Experimental] Whether to allow quoted newlines in the source CSV data. If this is set to FALSE, it allows for larger import sizes and may speed up import performance.
    configuration.load.sourceFormat string Optional and defaults to CSV. Format of source files. For CSV uploads, specify "CSV". For imports of datastore backups, specify "DATASTORE_BACKUP". For imports of newline-delimited JSON, specify "NEWLINE_DELIMITED_JSON".

    Extract Config

        "extract": {
              "sourceTable": {
                "projectId": string,
                "datasetId": string,
                "tableId": string
              },
              "destinationUri": string,
              "printHeader": boolean,
              "fieldDelimiter": string,
              "destinationFormat": string
            }
    Property Name Value Description
    configuration.extract nested object An object present if this is a job that exports a table to Google Cloud Storage.
    configuration.extract.sourceTable nested object An object describing the BigQuery table to be exported.
    configuration.extract.sourceTable.projectId string The ID of the project containing the source table.
    configuration.extract.sourceTable.datasetId string The datasetId of the dataset containing the source table. You must have read access on this dataset.
    configuration.extract.sourceTable.tableId string The tableId of the table to export, in the specified dataset.
    configuration.extract.destinationUri string The fully-qualified Google Cloud Storage URI of the object where you will save the table. This is in the format bucket/object with the gs:// prefix. You must have write access on this bucket. 

    Example:gs://mybucket/myobject


    To export to multiple files, provide a wild card in the URI: 

    gs://mybucket/myobject-*

    Which creates files named:

    gs://mybucket/myobject-0000
gs://mybucket/myobject-0001
gs://mybucket/myobject-0002
....
    configuration.extract.printHeader boolean [Optional] Whether to print out a heder row in the results. Default is true.
    configuration.extract.fieldDelimiter string [Optional] Delimiter to use between fields in the exported data. Default is ','
    configuration.extract.destinationFormat string Optional and defaults to CSV. Format with which files should be exported. To export to CSV, specify "CSV". Tables with nested or repeated fields cannot be exported as CSV. To export to newline-delimited JSON, specify "NEWLINE_DELIMITED_JSON".

    Query Config

       "query": {
      "query": string,
      "destinationTable": {
        "projectId": string,
        "datasetId": string,
        "tableId": string
      },
      "createDisposition": string,
      "writeDisposition": string,
      "defaultDataset": {
        "datasetId": string,
        "projectId": string
      },
      "priority": string
    }
    Property Name Value Description
    configuration.query nested object An object that must be present only when sending a query. If the query returns one or more result rows, results will be stored in the table specified by the destinationTable property.
    configuration.query.query string A query string, following the BigQuery query syntax of the query to execute. Table names should be qualified by dataset name in the formatprojectId:datasetId.tableId unless you specify the defaultDataset value. If the table is in the same project as the job, you can omit the project ID.

    Example:     SELECT f1 FROM myprojectId:myDatasetId.myTableId
    configuration.query.destinationTable nested object Describes the table where the query results should be stored. If not present, a new table will be created to store the results.
    configuration.query.destinationTable.projectId string [Required] ID of the project billed for storage of the table.
    configuration.query.destinationTable.datasetId string The datasetId of the dataset in which to create the result table. This is not qualified by the project ID. You must have write access on this dataset. If the specified dataset does not exist, the method will return an error.
    configuration.query.destinationTable.tableId string The tableId of the table to hold the results. This can be an existing table or not, depending on the query.createDisposition value.
    configuration.query.createDisposition string

    Whether to create a results table if no table by that ID already exists. The following string values are supported:

    • CREATE_NEVER - Do not create a new table to hold results.
    • CREATE_IF_NEEDED - [Default] Create a new table if one does not exist.
    configuration.query.writeDisposition string

    Whether or not to overwrite an existing results table with the specified name. The following string values are supported:

    • WRITE_EMPTY - [Default] Only write to a table with no data, otherwise fail.
    • WRITE_TRUNCATE - Clear the data from the existing table and append the output data. The table schema must match the schema of the new data.
    • WRITE_APPEND - Append the new data to any existing data. The table schema must match the schema of the new data.
    configuration.query.defaultDataset nested object Specifies the default datasetId and projectId to assume for any unqualified table names in the query. If not set, all table names in the query string must be fully-qualified in the formatprojectId:datasetId.tableid. You must specify either both datasetId and projectId, or neither (omit this object).
    configuration.query.defaultDataset.datasetId string The assumed dataset ID of any tables that are not qualified by dataset.
    configuration.query.defaultDataset.projectId string The assumed project ID of any tables not qualified by project.
    configuration.query.priority string [Optional] Specifies a priority for the query. Default is INTERACTIVE. Alternative is BATCH.

    Methods

    The following methods are supported:

    get
    Retrieves the specified job by ID. Note: Specify the unqualified jobId value, not the fully-qualified id value of the job.
    getQueryResults
    Returns query results from a query started by calling bigquery.jobs.query(). You can specify an optional wait time if the query is still running when the method is called; if the timeout expires before the query finishes, the method will return jobComplete=false,, and you will have to call it again to get query results. The maximum number of results that can be returned per page is 16,000 rows.
    insert
    Starts a new asynchronous job. You must have READ access to a project in order to run a job.
    list
    Lists all the Jobs in the current project started by the user calling this method. Jobs are retained indefinitely unless you call bigquery.jobs.delete on a job. Authentication is required.
    query
    Runs a query with an optional waiting period. The method will wait the specified waiting period to return; if the query has completed by then, the reply will include the first page of results; if the query has not completed by the specified waiting period, the method will return jobComplete=false, and you will have to call bigquery.jobs.getQueryResults() to get the query results. If you do not specify a waiting period, the method will hang until the query completes, or the job times out.
